# DroidHaz — 本格8方向リグ ＋ 武器の手統合（Option B）設計

作成: 2026-06-24 / ユーザー選択「B. 本格8方向リグ」。横/斜め/後ろの“偽装”をやめ、真の方向別ポーズにする。
あわせて武器の「第3の手」（独立した腕レイヤーが常に最前面）を解消する。

## 解く問題（根本原因）
- 現状は「1枚の正面ポーズ」を矩形/楕円で組み、向きは “顔パーツをずらす＋胴を傾ける＋左右反転” で偽装。
  → 正面の四角い頭は横顔にならず、横・斜めが「正面顔をずらしただけ」で違和感が出る（定数調整では直らない）。
- 武器は本体と独立の腕レイヤーを2本の腕の上に常時最前面で描画 → 手が3本・常に手前。

## 方向システム（既存を流用）
`js/render/facing.js` の `dirFromFacing()` → row ∈ {front, frontDiag, side, backDiag, back} ＋ flipX。
左右は flipX（鏡映ブリット）だけで担う＝右半面のみ作り込めばよい。

## リグの分離（refactor）
今の「キャラ別 body fn」を **共有リグ ＋ キャラ別スキン** に分ける（必要なら新モジュール `js/render/character-rig.js`）。
- **リグ（向き依存の骨格）**: 各 row について
  - 頭シルエットの形（front/back=正面四角、side=横顔シルエット＝前縁に鼻の出っ張り、frontDiag=やや片側を削る）
  - 胴の幅（side は細い、diag は中間）と肩の前後（diag は片肩前）
  - 腕アンカー＋「武器を持つ手＝前側の腕」＋奥腕の遮蔽（side は奥腕を省略 or 背後）
  - 脚の前後ずらし（side は前脚/後脚をスタッガー）
  - **武器ハンドのアンカー座標**と**武器の描画深度（前/後）**
- **スキン（キャラの中身）**: パレット、各 row の髪/帽子、装束、顔スタイル（human/skull/robot, bigEyes, brows 等）。
  10体（hero/mage/jk/skeleton/robot/acolyte/assassin/witch/swordmaster/mercenary）はスキンとして実装。

## 真の方向別ポーズ
- **side（真横）**: 頭は横顔シルエット（前縁に小さな鼻、目は1つ、髪は後頭に寄る）。胴は細く片肩。手前腕のみ（奥腕は背後/省略）。脚は前後スタッガー。
- **frontDiag（斜め前 3/4）**: 片肩を前に、両目（奥目は小さめ）、鼻は頭の前縁にわずかに。
- **backDiag（斜め後）**: ほぼ後頭部＋頬/耳のわずかな覗き。
- **front/back**: 既存をベースに（front は現状から大きく変えずリグに載せ替え＝退行を最小化）。

## 武器の手統合（第3の手の解消）
- 独立した武器腕レイヤーを**廃止**。
- キャラの「武器ハンド＝前側の腕」が照準方向へ伸びて武器を握る。もう一方の腕は体側に休める＝**腕は常に2本**。
- 武器は従来どおり照準角 `ang`（360°）に回転。ただし**原点＝手のアンカー**。
- **深度を向きで切替**:
  - 上/奥向き（aim.y < 0, back系）→ 武器＋腕は**胴の後ろ**に描く（背後で構える）。
  - 下/手前向き（aim.y > 0, front系）→ 胴の**前**。
  - side → 中間（手前腕が前、武器は体の前縁）。
- 近接スイングも同様に手のアンカー基準・深度切替。

## ピクセルアート（既存のドット格子）
- 現行の DOT 格子＋輪郭＋陰影のバッファ方式は維持し、**方向別ポーズをそのままバッファへ**描く（顔の可読性は DOT=4 で確保済み）。
